# Инструкция по работе с проектом

Данный документ содержит инструкцию по работе с проектом, размещённым на GitHub, и описывает его структуру.

## Как внести изменения в проект через веб-интерфейс GitHub

1. Перейти в репозиторий проекта.
2. Выбрать нужный файл.
3. Нажать на иконку карандаша (Edit this file).
4. Внести необходимые изменения в редакторе.
5. Внизу страницы указать сообщение коммита.
6. Нажать кнопку **Commit changes**.

## Структура проекта

- `README.md` — основная информация о проекте и цели лабораторной работы.
- `index.html` — главная HTML-страница сайта.
- `docs/instructions.md` — инструкция по работе с проектом.

## Работа с Issues

Для управления задачами используется вкладка **Issues**:
- создаются задачи по улучшению проекта;
- задачам можно назначать исполнителей и метки;
- Issue автоматически закрывается при указании `Closes #номер` в сообщении коммита.

## Работа с ветками и Pull Request

1. Для новой функциональности создаётся отдельная ветка.
2. Изменения вносятся в файлы в этой ветке.
3. После завершения работы создаётся Pull Request.
4. Pull Request проверяется и сливается с основной веткой `main`.

## Публикация сайта

Сайт публикуется с помощью GitHub Pages:
- используется ветка `main`;
- корневая папка `/`;
- после активации появляется ссылка на опубликованный сайт.
